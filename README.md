🌾 AgroChem Calculator: Автоматический подбор удобрений
Простой и эффективный консольный инструмент для автоматического расчета норм внесения минеральных удобрений и химических мелиорантов (извести). Программа принимает на вход результаты лабораторного анализа почвы и переводит сухие цифры в конкретные рекомендации: что купить, сколько килограмм внести на гектар и когда это сделать.

Проект разработан с прицелом на конечного пользователя (фермера или садовода), которому не нужно знать тонкости агрохимии — система всё посчитает сама.

🚀 Основные возможности
Индивидуальный подход к культурам: Разные растения требуют разного уровня питательных веществ. В базе уже заложены нормативы для озимой пшеницы, картофеля и томатов.

Умное определение дефицита: Система не просто говорит «мало азота», а оценивает степень голодания растений (небольшой, умеренный или острый дефицит).

Конвертация в «физический вес»: Агрохимики считают нормы в чистом действующем веществе (д.в.). Программа сама пересчитывает эти значения в реальный вес конкретного удобрения (например, Аммиачной селитры с учетом её процентного состава).

Раскисление почвы: если уровень pH ниже комфортного для культуры, система рассчитает необходимую дозу молотой извести для нейтрализации кислотности.

🧠 Как это работает (Агрохимия «на пальцах»)
Для управления урожаем программа отслеживает 4 ключевых параметра:

N (Азот): Отвечает за рост зеленой массы, стеблей и листьев.

P (Фосфор): Необходим для развития мощной корневой системы и формирования плодов.

K (Калий): Повышает иммунитет растений, устойчивость к засухе и улучшает вкусовые качества.

pH (Кислотность): Определяет, насколько комфортно корням «усваивать» элементы из почвы. Если почва слишком кислая, удобрения просто заблокируются и не принесут пользы.

Логика расчета дозировки:
Программа сравнивает ваш анализ с оптимальным минимумом для выбранной культуры.

Если обнаружен дефицит, рассчитывается процент отклонения и присваивается уровень критичности (1, 2 или 3)

Базовая доза элемента умножается на коэффициент культуры (ведь томатам нужно больше калия, а пшенице — азота)

Итоговый вес удобрения корректируется в зависимости от концентрации в нем активного вещества

🛠 Технологический стек
Язык программирования: Python 3.x

База данных: SQLite 3 (все справочники, нормы и доступные удобрения хранятся локально в файле agrochem.db)

1. Клонирование репозитория

💻 Пример работы программы
После запуска перед вами откроется интерактивное меню:

==================================================
  ПОДБОР УДОБРЕНИЙ ПО РЕЗУЛЬТАТАМ АНАЛИЗА ПОЧВЫ   
==================================================
  [1] Картофель
  [2] Пшеница озимая
  [3] Томат

Выберите номер культуры: 1

Введите результаты лабораторного анализа почвы:
  Азот N (мг/кг): 55
  Фосфор P2O5 (мг/кг): 65
  Калий K2O (мг/кг): 120
  Кислотность pH: 5.1

Результат расчета:

--- Рекомендации для культуры: Картофель ---
Нормативы оптимума: N(70-110), P(60-90), K(150-220), pH(5.5-6.5)

• Элемент [N] -> НЕБОЛЬШОЙ ДЕФИЦИТ:
  Рекомендуется: Аммиачная селитра — 105.9 кг/га
  Способ применения: основное (весна) (Аммонийно-нитратный азот)

• Элемент [K] -> УМЕРЕННЫЙ ДЕФИЦИТ:
  Рекомендуется: Сульфат калия — 252.0 кг/га
  Способ применения: предпосевное (Бесхлорное, идеально для овощей)

⚠️ ТРЕБУЕТСЯ ХИМИЧЕСКАЯ МЕЛИОРАЦИЯ (pH 5.1 при норме от 5.5):
  Рекомендуется: Известь молотая — 3.2 т/га (основное под вспашку)
==================================================

🗂 Структура базы данных
Для хранения агрономических правил используются три таблицы:

1. crops (Культуры)
Хранит системные коды растений и их понятные названия для интерфейса.

2. crop_norms (Агро-нормативы)

3. fertilizers (Каталог удобрений)
Содержит информацию о коммерческих марках удобрений, содержании в них действующего вещества (content_pct) и технологические заметки по внесению.
